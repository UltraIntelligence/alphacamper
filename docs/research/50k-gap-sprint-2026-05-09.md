# 50k Canada Gap Sprint - 2026-05-09

## Verdict

The smallest honest path across 50,000 is Manitoba Parks.

- Previous board-safe provider availability count: 44,817 campsite IDs.
- Gap to 50,000: 5,183 campsite IDs.
- Manitoba Parks live provider proof: 5,480 campsite IDs.
- Manitoba-only total: 50,297 campsite IDs.
- Manitoba plus Nova Scotia total: 51,997 campsite IDs.

This proves official/provider campsite inventory enumeration from live availability responses. It does not prove Railway heartbeat, active customer watch polling, or customer notification delivery.

## State Separation

| State | Meaning | Current read |
|---|---|---|
| Searchable | The campground can appear in customer search. | BC, Ontario, Parks Canada, New Brunswick, Manitoba, and Nova Scotia are already searchable in the live catalog. |
| Provider-countable | The provider availability response returns campsite-level IDs that can be counted without guessing. | Manitoba and Nova Scotia are now green from live provider availability proof. |
| Worker-pollable | The Railway worker has the domain and CAMIS polling path needed to check watches. | Manitoba and Nova Scotia use the existing GoingToCamp/CAMIS path in `DOMAINS` and `SUPPORTED_PLATFORMS`. |
| Realtime-alertable readiness | Repo metadata can honestly label the provider `alertable` and `live_polling` after provider proof. | Manitoba and Nova Scotia are upgraded in the worker catalog profile in this sprint. |
| Notification-proven | A real customer watch creates an alert and sends email/SMS/push. | Still yellow/blocked until Railway heartbeat and notification smoke are green. |

- `supportStatus: "alertable"`

## Remaining Yellow

Before customer-facing marketing can claim reliability:

- Railway worker heartbeat must be visible in `worker_status`.
- A real active watch must be polled by Railway.
- An alert row must be created from the worker path.
- Email/SMS/push notification delivery must be proven.
- Production catalog rows need a deploy/sync before the live app reflects the new Manitoba/Nova Scotia alertable labels.
- Provider health/admin view should show these providers clearly as live-polling, degraded, or blocked.
